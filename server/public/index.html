ChromaBridge is an enterprise-grade platform for real-time UI theming, design token distribution, and cross-application style orchestration.

Overview

ChromaBridge provides a centralized hub for managing design tokens — colors, typography, spacing, and component-level style overrides — across web applications, browser extensions, and native clients. Changes propagate in real-time via WebSocket, with REST endpoints for CRUD operations and batch syncing.

Quick Start

  • Install the SDK: npm i @chromabridge/sdk
  • Generate an API key in the Dashboard
  • Initialize the client with your org's endpoint
  • Subscribe to token change events
import { ChromaBridge } from '@chromabridge/sdk';

const cb = new ChromaBridge({
  endpoint: 'wss://api.chromabridge.io',
  apiKey: process.env.CB_API_KEY,
});

cb.on('tokens:updated', (tokens) => {
  applyTheme(tokens);
});

Authentication

All requests require an X-API-Key header. Keys are scoped per-organization and can be rotated in the dashboard.

curl -H "X-API-Key: cb_live_xxxx" \
  https://api.chromabridge.io/v1/tokens

WebSocket connections pass the key during the upgrade handshake. Unauthorized requests receive 401 Unauthorized.

REST API Reference

Base URL: https://api.chromabridge.io/v1

GET /tokens — List all design tokens for the authenticated organization.

POST /tokens — Create or update a batch of design tokens.

GET /tokens/:id — Retrieve a single token by ID.

GET /themes — List all theme configurations.

POST /themes/:id/apply — Push a theme to all connected clients.

WebSocket Events

Connect to wss://api.chromabridge.io/ws with your API key header.

WS tokens:updated — Emitted when design tokens change. Payload contains the full token set.

WS theme:applied — Emitted when a theme is pushed. Includes theme ID and resolved token values.

WS config:sync — Bi-directional config synchronization heartbeat.

// Client → Server (heartbeat)
{ "type": "TYPE_HEARTBEAT", "sessionId": "...", "timestamp": 1711234567890 }

// Server → Client (token update)
{ "type": "TYPE_UI_CONFIG", "theme": { "primary": "#7c3aed" } }

Rate Limits

REST endpoints: 1,000 requests/minute per API key. WebSocket connections: 100 messages/second per session. Exceeding limits returns 429 Too Many Requests.
